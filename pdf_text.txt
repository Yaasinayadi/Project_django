Partie 1 : Mise en place de l'environnement et du projet 
Avant de commencer le développement, nous devons préparer un environnement de travail 
propre et isolé, puis créer la structure de base du projet Django. 
1.1 Création de l'environnement virtuel 
Un environnement virtuel permet d'isoler les dépendances du projet et d'éviter les conflits 
entre différentes versions de bibliothèques. Ouvrez un terminal dans le dossier de votre choix 
et exécutez la commande suivante : 
python -m venv monenv 
Activez ensuite l'environnement virtuel. Sur Windows : 
monenv\Scripts\activate 
Une fois activé, votre invite de commandes affichera le préfixe (monenv) devant le chemin 
courant, indiquant que vous travaillez bien dans l'environnement isolé. 
1.2 Installation de Django 
Django est le framework web Python que nous allons utiliser. Avec l'environnement virtuel 
activé, installez Django via pip : 
pip install django 
Pip télécharge automatiquement Django et toutes ses dépendances. Le message « Successfully 
installed django » confirme le succès de l'installation. 
1.3 Création du projet Django 
Dans Django, un projet représente l'ensemble de votre site web et contient la configuration 
globale. Nous allons nommer notre projet « school ». Exécutez : 
django-admin startproject school 
cd school 
La commande django -admin startproject génère automatiquement un dossier school/ 
contenant le fichier manage.py et un sous -dossier school/ avec les fichiers de configuration 
(settings.py, urls.py, wsgi.py, asgi.py). Université Abdelmalek Essaadi

3.4 Intégration des templates HTML 
Copiez les fichiers HTML fournis (base.html et les templates de pages) dans le dossier 
templates/. Pour que Django puisse générer les URLs des fichiers statiques, ajoutez la balise 
{% load static %} en tête de chaque template qui utilise des fichiers stati ques, juste après la 
balise <!DOCTYPE html>. Remplacez également tous les liens vers des fichiers CSS/JS/images 
par la syntaxe Django : 
 
{% load static %} 
<!DOCTYPE html> 
<html lang='fr'> 
<head> 
  <!-- Avant : href='assets/css/style.css' --> 
  <!-- Après : --> 
  <link rel='stylesheet' href="{% static 'assets/css/style.css' %}"> 
</head> 
Faites de même pour tous les fichiers JS et les balises <img src='...'> dans vos templates.

Note : OneToOneField 
La relation OneToOneField entre Student et Parent signifie qu'un étudiant possède exactement un 
Parent, et un Parent est lié à exactement un étudiant. Le paramètre on_delete=models.CASCADE 
indique que si le parent est supprimé, l'étudiant associé l'est également.
